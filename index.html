Introduction

I'm a 28-year-old Business Analyst with a love for data, who firmly believes in automation (especially in enviroments that aren't optimized for it). My main beliefs are those of information sharing (for example a central knowledge hub), finding solutions for problems, not the people who caused it (pointing fingers), and actionable reporting (don't make dashboards just because you like looking at them).

Reports and dashboards have to have a purpose and be actionable. There's no reason to have hundredrs of dashboard when no one knows what to do with it or how to interpret it.

When there are repetitive manual tasks happening, first you have to wonder if it's even worth doing them, secondly you have to automate them as much as possible with the tools at your disposal. Every repetitive task that has to be done manually are hours that are lost for no good reason. The snippets on this website might help you in enviroments that aren't optimzed for automation.

Code Snippets

Because I want to help others automate (boring) tasks, I've listed some snippets on a separate page. These aren't perfect snippets, nor do I pretend they are. My main intention is to give examples to others as to what they can automate and how they could possible go around doing so.

These automations are written in VBA, PowerShell and Power Query M. They aren't meant as a way to show off my skills - as some of them are even written by others. A lot of these snippets wouldn't even be necessary with a proper data sharing platform, but sometimes you have to work with what you got.

Skills

  • DAX
    The last few years I've been building a lot of reports in Power BI (and Excel - Power Pivot) whereby knowledge of DAX is crucial. Especially when it comes to building measures that are easy to maintain and are easily used withing time intelligence.
  • Power Query M
    Crucial part of loading data into your data model in Excel and Power BI. Knowing how to write M and not just using the GUI can make a massive difference in performance. Understanding what you're doing and using loops, variables, etc. where needed will help you enormously.
  • PowerShell
    Mostly learned to use this in a nonorthodox way, because other alternatives weren't available. Using COM objects to interact with Office, I've been able to automate a plethora of otherwise boring, repeating tasks.
  • Python
    It has been a while since I've used Python in a professional context (mostly due to limitations), but I have a broad basis in Python (used it in University) and I know how to use pandas to do data analysis and manipulation.
  • VBA
    This might not be loved as much as it used to, but it still has very useful tricks up its sleeves. In combination with PowerShell you can make very powerful, user-agnostic scripts.
